# AI Platform 业务需求文档 (Business Requirements)

**角色**: 业务部门负责人
**日期**: 2026-05-07
**版本**: v1.0

---

## 一、需求背景

作为业务部门负责人，我关心的是：
1. AI服务是否稳定可用？
2. 成本花了多少、能否给客户计费？
3. 出了问题能否快速恢复？
4. 业务人员能否轻松使用，不需要写代码？

---

## 二、功能需求

### 2.1 业务驾驶舱（P0）

**目标**: 一页看清所有关键业务指标，不需要跳转多个菜单。

**功能要求**:
| 功能 | 说明 |
|------|------|
| 总览卡片 | 今日调用量、在线Agent数、平均响应时间、今日成本 |
| 趋势图表 | 24小时调用量曲线、响应时间分布、成本趋势 |
| 异常标记 | 问题指标标红（如响应超时率>5%） |
| 快捷入口 | 直接跳转到问题Agent详情 |

**使用场景**:
- 每天早上5分钟扫一眼驾驶舱，了解昨日整体情况
- 发现异常立刻定位到具体Agent

**验收标准**:
- [ ] 首屏加载 < 2秒
- [ ] 数据实时刷新（30秒间隔）
- [ ] 支持全屏展示（大屏场景）

---

### 2.2 成本与计费中心（P0）

**目标**: 清楚知道AI钱花在哪里，能否给客户计费。

**功能要求**:
| 功能 | 说明 |
|------|------|
| Token消费统计 | 按Agent、业务线、客户多维度统计 |
| 成本趋势 | 日/周/月成本曲线，支持对比 |
| 预算预警 | 设置预算阈值，超出时告警 |
| 账单导出 | 支持Excel导出，便于财务对账 |

**使用场景**:
- 月底给财务提交AI费用报告
- 发现某业务线成本异常高，追查原因

**验收标准**:
- [ ] 支持按时间/Agent/业务线筛选
- [ ] 账单数据可导出
- [ ] 预算告警可配置

---

### 2.3 告警与应急响应（P0）

**目标**: 问题第一时间知道，能快速恢复。

**功能要求**:
| 功能 | 说明 |
|------|------|
| 告警规则 | 可配置阈值：错误率、响应时间、掉线率 |
| 通知渠道 | 支持钉钉群、邮件、Webhook |
| 告警级别 | P0紧急 / P1重要 / P2提示 |
| 一键回滚 | Agent版本回退，不需要技术介入 |

**使用场景**:
- Agent掉线了，我立刻收到钉钉通知，点链接看详情
- 发现新版本有问题，点击"回滚到上一版本"

**验收标准**:
- [ ] 告警延迟 < 1分钟
- [ ] 支持钉钉/邮件双渠道
- [ ] 回滚操作 < 3分钟完成

---

### 2.4 操作审计日志（P1）

**目标**: 满足合规要求，能追溯每一笔操作。

**功能要求**:
| 功能 | 说明 |
|------|------|
| 操作日志 | 记录：用户、 时间、Agent、输入、输出、耗时 |
| 日志查询 | 按时间/用户/Agent/关键词筛选 |
| 日志导出 | 支持导出用于审计 |
| 合规报表 | 月度/季度操作汇总报表 |

**使用场景**:
- 客户问"我的数据谁访问了？"，我能查到
- 审计时能提供完整的操作记录

**验收标准**:
- [ ] 日志保留至少6个月
- [ ] 查询响应 < 3秒（支持千万级数据）
- [ ] 支持SQL导出

---

### 2.5 客户账户管理（P1 - 对外服务场景）

**目标**: 向外部客户提供AI服务时，能管理账户和计费。

**功能要求**:
| 功能 | 说明 |
|------|------|
| 客户账户 | 创建/冻结/删除客户账户 |
| 余额管理 | 充值、扣费、余额预警 |
| 客户账单 | 按客户生成月度账单 |
| 欠费策略 | 余额不足时暂停服务 |

**使用场景**:
- 新客户注册，充值后开通服务
- 客户余额不足，系统自动暂停服务并通知

**验收标准**:
- [ ] 客户自助充值
- [ ] 余额变动有记录
- [ ] 欠费自动处理

---

### 2.6 低代码调用界面（P1）

**目标**: 业务人员不需要写代码，也能调用Agent。

**功能要求**:
| 功能 | 说明 |
|------|------|
| 可视化调用 | 选择Agent → 填写参数 → 执行 |
| 预置模板 | 常见场景一键调用（如：图像识别、TTS播报） |
| 执行记录 | 历史调用可追溯、可重试 |
| 结果预览 | 执行结果直接展示，支持图片/音频预览 |

**使用场景**:
- 客服问我"能不能批量识别这批图片"，我来这里配置一下执行

**验收标准**:
- [ ] 无代码基础也能使用
- [ ] 支持批量执行
- [ ] 结果可直接下载

---

### 2.7 移动端支持（P2）

**目标**: 手机上能看数据和审批。

**功能要求**:
| 功能 | 说明 |
|------|------|
| 驾驶舱移动版 | 核心指标移动端展示 |
| 告警通知 | 手机收到推送，点击查看详情 |
| 审批流程 | 简单审批操作可在手机完成 |

**使用场景**:
- 地铁上收到告警，点开看具体情况
- 需要审批一个预算调整，在手机上操作

**验收标准**:
- [ ] 支持iOS/Android
- [ ] 主要数据可看
- [ ] 告警可推送

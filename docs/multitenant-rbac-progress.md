## 已完成模块总览

### 多租户与 RBAC

- 新增租户权限底座：租户、用户租户关系、用户角色、角色权限、菜单等实体、Mapper、服务和管理接口。
- 新增默认租户 `Think Land`，历史用户和业务数据迁入默认租户。
- 初始化平台管理员、租户管理员、开发者、审核员、只读成员等内置角色。
- 初始化权限码，覆盖 Agent、Skill、RAG、模型、流水线、代码质量、AI 治理、提示词工程、账单、审计、租户、成员、角色权限、菜单权限等模块。
- 登录态增加租户、角色、权限、平台管理员信息。
- 登录响应返回当前租户、可切换租户、权限码和菜单树。
- 新增 `TenantContext`，从 JWT 注入请求租户上下文，请求结束后清理。
- 新增 `@RequirePermission` 注解和 AOP 权限校验，主要管理类接口已加权限码。
- 接入 MyBatis-Plus 租户拦截器，业务表按 `tenant_id` 自动过滤。
- 前端路由增加权限码校验，无权限进入 `403` 页面。
- 顶部增加当前租户显示和租户切换入口。

### 后台菜单与权限配置

- 侧边栏支持后端菜单树。
- 菜单支持一级/二级结构和折叠状态。
- 系统管理入口包括：
  - 租户管理
  - 成员管理
  - 角色权限
  - 菜单权限
- 权限配置路径：
  - `成员管理`：给用户加入租户，并分配角色。
  - `角色权限`：给角色勾选接口权限码。
  - `菜单权限`：维护菜单入口、父级菜单和菜单绑定权限码。
  - `租户管理`：维护租户基础信息。

### Agent 调用图谱

- 图谱页面已支持在已注册且存在的 Agent 节点之间直接拖拽连线。
- 连线后调用后端评估接口，提示当前关联是否合适。
- 创建关联不再使用填空式弹窗作为主流程。
- 修复过 `/api/v1/agent-graph/edges/evaluate` 的权限放行问题，避免拖拽评估出现 403。
- 关联评估提示已改为中文展示。

### Skill 管理

- Skill 管理页已增加 AI 生成提示能力。
- 支持在模型协助下生成 Skill 草稿，帮助不熟悉 Skill 编写的用户创建自己的 Skill。

### RAG 混合切分

- 后台 RAG 已规划并实现混合切分能力。
- 支持识别文章标题，保留标题上下文。
- 支持针对表格识别表格标题，降低切分表格时的上下文丢失和内容污染风险。

### 自动化流水线

- 流水线详情已改为中文介绍。
- 增加概况说明。
- 流水线阶段文案从英文阶段名调整为中文业务说明。

### 代码质量评估

- 自动化流水线新增代码质量评估标准版能力。
- 创建流水线支持单独选择代码评估模型。
- `AutomationPipelineRequest` 增加 `qualityModelCode`。
- `automation_pipeline` 增加 `quality_model_code` 字段。
- 代码生成模型和代码评估模型拆开配置。
- 启用代码质量评估但未选择评估模型时，后端返回参数错误。
- 评估维度包括：需求符合度、可运行性、安全性、架构合理性、可维护性、可读性、测试完整性、性能风险。
- 质量门禁支持总分、安全分、需求符合度、BLOCKER/CRITICAL/MAJOR 数量阈值。
- 门禁不通过时质量阶段置为 `BLOCKED`，阻塞后续构建编译阶段。
- 前端支持展示总分、维度分、问题列表、阻塞状态和重新评估。

### AI 产出治理

- 已向 Harness 风格治理架构靠近。
- 治理记录与人工审批、质量门禁打通，逐步形成真正的放行链路。

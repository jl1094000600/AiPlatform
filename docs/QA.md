## 六、AI 产品经理专项类

### 6.1 AI 基础认知

| 编号 | 问题 | 参考答案 | 难度 |
|------|------|----------|------|
| AI001 | 大模型（LLM）和传统机器学习模型的区别是什么？ | 传统模型需要大量标注数据训练，针对单一任务；LLM 采用预训练 + 微调范式，具备涌现能力，可处理多模态、多任务。 | ★★☆ |
| AI002 | 什么是 RAG？它在 AI Platform 中如何应用？ | Retrieval-Augmented Generation，检索增强生成。通过外挂知识库解决大模型幻觉问题，提升回答准确性。项目中可结合 Agent 知识查询场景。 | ★★★ |
| AI003 | 什么是 Token？为什么它影响大模型的成本和性能？ | Token 是文本处理的最小单位（中文约 2-3 字符 = 1 Token）。大模型按 Token 计费，上下文越长计算成本越高、延迟越大。 | ★★★ |
| AI004 | Agent 和普通 API 调用的本质区别是什么？ | Agent 具备自主推理、工具调用、多轮对话记忆能力；普通 API 是确定性函数调用，输入输出固定。Agent 更适合复杂任务分解和动态决策。 | ★★★ |
| AI005 | 什么是多模态 Agent？项目中哪些 Agent 属于多模态？ | 能处理多种模态（文本、图像、语音）的 Agent。图像识别 Agent 属于多模态，处理图像输入 + 文本输出；TTS Agent 属于语音生成。 | ★★☆ |
| AI006 | 什么是 Prompt Engineering？为什么重要？ | 通过优化提示词引导大模型输出更准确结果的技术。包括角色设定、few-shot examples、输出格式约束等。产品经理需掌握以提升 AI 产品输出质量。 | ★★★ |

### 6.2 AI 产品设计

| 编号 | 问题 | 参考答案 | 难度 |
|------|------|----------|------|
| AI007 | 设计一个 AI 助手类产品，核心要考虑哪些要素？ | 1. 明确目标用户和使用场景；2. 定义 AI 能力边界（什么能做、什么不该做）；3. 设计人机协作流程；4. 设置护栏防止滥用；5. 定义效果评估指标。 | ★★★ |
| AI008 | 如何评估 AI 产品的效果？有哪些常用指标？ | 1. 准确性指标（Precision/Recall/F1）；2. 效率指标（响应时间、吞吐量）；3. 用户满意度（NPS、Task Completion Rate）；4. 业务指标（转化率、成本节省）。AI 产品还需关注「幻觉率」。 | ★★★ |
| AI009 | AI 产品的用户体验和传统产品有什么不同？ | 1. AI 输出不确定性，需设计容错机制和置信度展示；2. 交互模式更自然（对话式），但需管理用户预期；3. 需要教育用户如何有效提问；4. 隐私和数据安全更敏感。 | ★★★ |
| AI010 | 什么是 AI 产品的「幻觉」问题？如何缓解？ | 幻觉指大模型生成看似合理但实际错误的内容。缓解方法：1. RAG 外挂知识库；2. 设置置信度阈值；3. 增加人工审核环节；4. 优化 Prompt 约束输出格式。 | ★★★ |
| AI011 | 对话式 AI 产品如何设计多轮对话记忆？ | 1. 短期记忆：对话上下文窗口；2. 长期记忆：用户画像、偏好持久化；3. 结构化存储：关键信息提取到数据库；4. 向量检索：语义相似历史召回。 | ★★★ |
| AI012 | 如果用户反馈 AI 回复质量差，如何定位问题？ | 1. 区分「模型问题」还是「Prompt 问题」；2. 检查知识库覆盖度；3. 统计高频错误类型；4. A/B 测试不同模型/Prompt；5. 收集用户反馈持续优化。 | ★★★ |

### 6.3 AI 业务落地

| 编号 | 问题 | 参考答案 | 难度 |
|------|------|----------|------|
| AI013 | 什么样的场景适合用 AI Agent 来解决？ | 适合场景：1. 多步骤复杂任务（需分解执行）；2. 需要调用多个工具/API；3. 动态决策依赖上下文；4. 需要长期记忆积累。不适合：简单 CRUD、严格确定性要求的任务。 | ★★★ |
| AI014 | 如何判断一个业务场景是否适合 AI 化改造？ | 评估框架：1. 业务价值（频率 × 影响 × 风险）；2. AI 可行性（数据充分性 × 任务复杂度 × 容错空间）；3. 改造成本（数据准备 × 集成难度 × 维护成本）。 | ★★★ |
| AI015 | AI 产品上线后，需要持续关注哪些运营指标？ | 1. 技术指标：响应延迟、错误率、Token 消耗；2. 效果指标：准确率、用户满意度；3. 业务指标：任务完成率、转化率变化；4. 成本指标：单次调用成本、资源利用率。 | ★★★ |
| AI016 | 如何控制 AI 产品的使用成本？ | 1. 模型选型：按任务复杂度分级，简单任务用小模型；2. Prompt 优化：减少无效 Token；3. 缓存策略：相同问题复用结果；4. 限流策略：防止异常调用；5. 监控预警：成本超限告警。 | ★★★ |
| AI017 | 市场营销 Agent 为什么选择「销售查询、趋势分析、统计排名」这三个功能？ | 这三个功能具备高频、标准化、数据驱动的特点，适合 AI 赋能：1. 销售查询是日常高频需求；2. 趋势分析需要跨数据源关联，AI 能自动聚合；3. 统计排名规则明确，适合 AI 快速输出。 | ★★☆ |

### 6.4 AI 技术理解

| 编号 | 问题 | 参考答案 | 难度 |
|------|------|----------|------|
| AI018 | SpringAI 在项目中的作用是什么？ | Spring AI 是 Spring 生态的 AI 集成框架，简化与大模型（OpenAI、Azure OpenAI 等）的交互。项目通过 ChatModelService 调用 AI 能力，通过 AgentRegistry 管理 Agent 注册。 | ★★☆ |
| AI019 | 为什么 TTS 选择 Edge TTS 而不是自建语音合成？ | Edge TTS 是微软提供的云服务，延迟低、音质好、支持多种音色选择，且成本远低于自建。适合作为平台能力快速集成。 | ★★☆ |
| AI020 | 什么是向量数据库？为什么 Agent 需要它？ | 向量数据库存储 embedding 向量，支持语义相似性检索。Agent 可通过向量检索实现知识库问答、历史对话召回，提升回答相关性。 | ★★★ |
| AI021 | 流式输出（SSE）对 AI 产品体验有什么意义？ | 1. 降低感知延迟，用户无需等待完整结果；2. 提升交互自然感；3. 支持实时展示 AI 思考过程；4. 长文本场景用户体验提升明显。 | ★★☆ |
| AI022 | A2A 协议和多 Agent 协作是什么关系？ | A2A（Agent-to-Agent）协议定义了 Agent 之间的通信标准，多 Agent 协作通过 A2A 实现任务分发、结果汇聚。例如图像识别 Agent + TTS Agent 串联完成「看图说话」场景。 | ★★★ |

### 6.5 AI 产品伦理与风险

| 编号 | 问题 | 参考答案 | 难度 |
|------|------|----------|------|
| AI023 | AI 产品可能涉及哪些合规风险？ | 1. 数据隐私（用户数据用于模型训练）；2. 内容安全（生成有害/违规内容）；3. 知识产权（训练数据版权）；4. 算法公平性（歧视性输出）；5. 可解释性（决策过程不透明）。 | ★★★ |
| AI024 | 如何在 AI 产品中设置安全护栏？ | 1. 输入过滤：敏感词检测、注入攻击防护；2. 输出过滤：违禁内容识别、人工抽检；3. 权限控制：角色权限分级、数据隔离；4. 审计日志：完整操作记录追溯。 | ★★★ |
| AI025 | 产品经理如何参与 AI 伦理治理？ | 1. 在 PRD 中定义「不该做什么」；2. 设计人工介入机制（如敏感内容需审核）；3. 建立用户反馈和投诉通道；4. 关注监管政策动态（如《生成式AI服务管理暂行办法》）。 | ★★☆ |

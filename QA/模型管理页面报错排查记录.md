# 模型管理页面报错排查记录

## 基本信息
- 时间：2026-05-07
- 页面：模型管理
- 现象：页面顶部提示“加载模型列表失败”，模型数量显示为 0。
- 团队模式：
  - 项目经理：记录问题、影响范围和后续处理项。
  - 产品经理：验收口径为模型管理页可正常加载模型列表或展示空状态。
  - 架构师：检查前端 API、Spring Security、MVC 拦截器、模型服务链路。
  - 前端：确认页面调用 `GET /api/v1/models`，失败后弹出错误提示。
  - 后端：确认模型接口存在，但鉴权链路未放行。
  - 测试：完成接口级复现；Redis 依赖不参与本问题判断。

## 复现与证据
1. 前端 `front/src/views/ModelList.vue` 在页面挂载时调用 `api.getModels({ pageNum, pageSize })`。
2. `front/src/api/index.js` 将该请求映射到 `GET /api/v1/models`。
3. 后端 `ModelController` 已提供 `GET /api/v1/models`，`ModelService` 查询 `ai_model` 分页列表。
4. 直接请求 `http://localhost:8080/api/v1/models` 返回 `403`，响应体为空。
5. 对照请求：
   - `GET /api/v1/agents` 返回 `200`
   - `GET /api/v1/monitor/agent-graph` 返回 `200`
   - `GET /api/v1/models` 返回 `403`
6. `SecurityConfig` 当前只放行了 `/api/v1/auth/**`、`/api/v1/agents/**`、`/api/v1/registry/agents/**`、`/api/v1/monitor/**`、`/api/v1/heartbeat/**` 等路径，未放行 `/api/v1/models/**`。
7. `InterceptorConfig` 当前也未将 `/api/v1/models` 纳入免 JWT 白名单。

## 根因
模型管理页失败的根因是权限白名单缺失。模型接口本身存在，前端调用路径也正确，但 `/api/v1/models` 没有被 Spring Security 和 MVC JWT 拦截器放行，未携带有效认证时被拒绝，导致前端进入 catch 分支并提示“加载模型列表失败”。

## 影响范围
- 影响：模型管理页列表加载、新增/编辑/删除模型接口在未认证或白名单模式下都会失败。
- 不影响：Agent 管理、调用图谱、心跳、注册接口当前已能访问。
- Redis：无关。本问题发生在 HTTP 鉴权层，未进入模型服务或数据库逻辑。

## 建议处理
- 后端将 `/api/v1/models/**` 加入 `SecurityConfig` 放行列表。
- 后端将 `/api/v1/models` 和 `/api/v1/models/**` 加入 `InterceptorConfig` 白名单，保持与 Agent/Monitor 页面一致。
- 如产品希望模型管理必须登录访问，则前端需要在请求失败时展示 401/403 的权限提示，并确认登录态 token 是否真实有效；当前项目其他管理页已采用白名单模式，因此推荐本次与现有页面保持一致。

## 测试与验收
- 接口测试：
  - 修复前：`GET /api/v1/models` 返回 `403`。
  - 修复后预期：`GET /api/v1/models` 返回 `200`，响应结构为 `{ code, message, data.records, data.total }`。
- 产品验收：
  - 模型为空时展示“暂无模型”空状态，不弹错误提示。
  - 有模型数据时展示模型卡片、总数和分页。
  - 新增、编辑、删除模型操作不因鉴权白名单缺失失败。

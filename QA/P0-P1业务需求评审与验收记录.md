# P0+P1 业务需求评审与验收记录

## 基本信息
- 时间：2026-05-07
- 输入文档：`marketDoc/business-requirements.md`
- 范围：P0 + P1，排除 P2 移动端
- 团队模式：业务部门负责人、产品经理、项目经理、架构师、前端、后端、测试

## 需求评审结论
- 业务部门负责人：本次必须形成“能看、能管、能追责、能计费、能给业务人员使用”的业务闭环。
- 产品经理：P0 与 P1 全部纳入本次开发；各模块独立验收，最终做 UAT。
- 项目经理：拆成 M1 驾驶舱、M2 计费与客户、M3 告警与审计、M4 低代码调用四个阶段推进。
- 架构师：复用现有 Agent、Monitor、Audit、Workflow、A2A、Model 基础，不重做平台底座。

## 已实现模块
- 业务驾驶舱：新增 `/dashboard` 页面和 `/api/v1/business-dashboard/*` 聚合接口。
- 成本与计费：新增 `/billing` 页面和用量、趋势、预算、账单导出接口。
- 告警与应急响应：新增 `/alerts` 页面，支持告警规则、事件、确认和手动检测。
- 操作审计日志：新增 `/audit-logs` 页面，支持查询和导出。
- 客户账户管理：新增 `/customers` 页面，支持创建、编辑、冻结和余额调整。
- 低代码调用界面：新增 `/invoke` 页面，支持选择 Agent、JSON 参数执行、历史、重试和下载。

## 测试与验收
- 后端测试：新增驾驶舱聚合、告警阈值、客户余额服务测试。
- 前端测试：新增业务运营菜单与基础逻辑测试。
- 静态检查：`git diff --check` 用于提交前检查。
- 产品 UAT 场景：
  - 早会打开业务驾驶舱查看今日调用量、在线 Agent、成本和异常。
  - 财务导出指定时间范围账单。
  - 运维配置错误率告警并触发事件确认。
  - 业务冻结欠费客户并记录余额流水。
  - 无代码选择 Agent 执行一次调用并下载结果。

## 风险与约束
- 成本按 Token 固定单价估算，未接云厂商真实账单。
- 回滚为 Agent 版本状态回退，不做容器或部署系统回滚。
- 通知渠道先记录配置和告警事件，真实钉钉/邮件/Webhook 发送可在下一阶段增强。
- Redis 相关不作为本次验收阻塞项。

# AI Platform 项目亮点 - 简历用

> 整理自 AI Platform v3.0 项目，从产品经理、AI产品经理、Java开发三个视角提取简历项目亮点。

---

## 一、产品经理视角

### 1.1 产品规划与版本迭代

| 亮点描述 | 具体说明 |
|---------|---------|
| **版本迭代管理** | 主导 AI Platform 从 v1.0 到 v3.0 的演进，v3.0 新增图像识别Agent、市场营销Agent、TTS Agent三大垂直领域能力，完成前端科技美学改造 |
| **6周迭代周期** | 采用6周为一个迭代周期，平衡开发效率与交付质量，关键里程碑：M1 PRD评审 → M2 后端开发 → M3 前端改造 → M4 集成联调 → M5 测试修复 → M6 发布上线 |
| **PRD规范化输出** | 输出完整PRD文档，包含功能描述、业务流程、接口设计、验收标准，确保开发、测试、UI多方对齐 |

### 1.2 需求分析与优先级管理

| 亮点描述 | 具体说明 |
|---------|---------|
| **需求价值评估** | 建立业务价值（频率×影响×风险）与AI可行性（数据充分性×任务复杂度×容错空间）评估框架，判断业务场景是否适合AI化改造 |
| **垂直领域Agent规划** | 设计图像识别Agent（视觉理解型）、市场营销Agent（数据分析型）、TTS Agent（语音合成型）三大类型，定义各自核心能力与协作关系 |
| **优先级决策** | 协调多角色（前端、UI、UX、后端）工作，明确各角色交付物和截止时间，关键节点评审对齐 |

### 1.3 用户体验设计

| 亮点描述 | 具体说明 |
|---------|---------|
| **数据驾驶舱风格** | 将仪表盘打造为数据驾驶舱，统计卡片突出关键数字，配合动效提升科技感和专业感，便于管理者一目了然看到全局状态 |
| **科技美学设计规范** | 定义色彩体系（深蓝黑#0a0e27 + 霓虹蓝#00d4ff/紫#7c3aed）、玻璃拟态卡片、粒子动画背景、数字滚动计数等UI规范 |
| **图谱可视化** | 设计Agent调用关系图谱，支持节点拖拽、实时状态、流动连线动画，提升复杂协作网络的理解效率 |

### 1.4 项目管理

| 亮点描述 | 具体说明 |
|---------|---------|
| **跨团队协作** | 协调产品、设计、前端、后端、测试多方资源，建立定期同步机制，关键路径功能优先开发 |
| **风险预控** | 提前识别数据源风险（市场营销Agent依赖销售数据）、多模态模型风险（图像识别依赖SpringAI Vision能力）并制定应对方案 |
| **发布节奏把控** | 制定详细发布计划，包含数据库迁移、端口变更、配置更新等注意事项，确保平滑升级 |

## 二、AI产品经理视角

### 2.1 AI能力规划

| 亮点描述 | 具体说明 |
|---------|---------|
| **Agent类型定义** | 设计三类垂直领域Agent：图像识别Agent（多格式识别+文档提取）、市场营销Agent（销售查询+趋势分析+统计排名）、TTS Agent（文本转语音+多音色+流式输出） |
| **A2A协作协议** | 设计Agent-to-Agent通信协议，支持多Agent协作任务分解，定义invoke/respond/delegate/broadcast四种Action类型 |
| **AI产品效果评估** | 建立准确性指标（Precision/Recall/F1）、效率指标（响应时间、吞吐量）、用户满意度（NPS、Task Completion Rate）、业务指标（转化率、成本节省）多维评估体系 |

### 2.2 AI技术理解

| 亮点描述 | 具体说明 |
|---------|---------|
| **SpringAI集成** | 理解SpringAI统一抽象的AI模型调用接口，支持OpenAI、Azure OpenAI、Anthropic等多供应商切换 |
| **RAG应用场景** | 了解检索增强生成（RAG）解决大模型幻觉问题，项目中应用于Agent知识查询场景 |
| **Token成本控制** | 理解Token是文本处理最小单位（中文约2-3字符=1Token），大模型按Token计费，上下文越长成本越高 |

### 2.3 AI产品设计

| 亮点描述 | 具体说明 |
|---------|---------|
| **Agent vs API本质区别** | Agent具备自主推理、工具调用、多轮对话记忆能力；普通API是确定性函数调用，输入输出固定 |
| **多模态Agent设计** | 设计能处理多种模态（文本、图像、语音）的Agent，图像识别Agent属于多模态（图像输入+文本输出），TTS Agent属于语音生成 |
| **对话式AI记忆设计** | 设计短期记忆（对话上下文窗口）和长期记忆（用户画像、偏好持久化）机制 |

### 2.4 AI伦理与安全

| 亮点描述 | 具体说明 |
|---------|---------|
| **安全护栏设计** | 设计输入过滤（敏感词检测、注入攻击防护）、输出过滤（违禁内容识别）、权限控制（角色权限分级）、审计日志（完整操作记录追溯） |
| **幻觉问题应对** | 设计RAG外挂知识库、置信度阈值设置、人工审核环节、Prompt约束输出格式等多种缓解方案 |
| **合规风险识别** | 识别数据隐私、内容安全、知识产权、算法公平性、可解释性等AI产品合规风险点 |

### 2.5 AI落地评估

| 亮点描述 | 具体说明 |
|---------|---------|
| **场景选择框架** | 适合AI场景：多步骤复杂任务（需分解执行）、需要调用多个工具/API、动态决策依赖上下文、需要长期记忆积累 |
| **成本控制策略** | 模型选型按任务复杂度分级、Prompt优化减少无效Token、缓存策略复用结果、限流策略防止异常调用 |
| **运营指标体系** | 技术指标（响应延迟、错误率、Token消耗）、效果指标（准确率、用户满意度）、业务指标（任务完成率、转化率）、成本指标（单次调用成本、资源利用率） |

## 三、Java开发视角

### 3.1 架构设计

| 亮点描述 | 具体说明 |
|---------|---------|
| **Spring Boot 3.2 + JDK 21** | 采用最新Spring Boot 3.2和JDK 21，利用虚拟线程（Virtual Threads）提升并发处理能力 |
| **微服务架构** | 采用多模块项目结构，主平台（8080端口） + 图像识别Agent（8082端口）+ 市场营销Agent（8081端口），独立部署、独立扩展 |
| **SpringAI集成** | 集成SpringAI 1.0.5，统一抽象ChatModel调用，支持OpenAI、Azure OpenAI、Anthropic等多供应商，通过ChatModelService动态加载模型配置 |

### 3.2 核心功能实现

| 亮点描述 | 具体说明 |
|---------|---------|
| **A2A通信协议** | 基于Redis Stream实现Agent之间异步通信，支持消息持久化、消费组模式保证顺序，通过A2AMessageService管理消息发送、响应获取、处理器注册 |
| **Agent心跳机制** | 实现Push+Pull双通道注册（Agent自注册+平台定时探测），心跳间隔30秒，超时90秒判定离线，使用Redis Hash存储心跳状态，定时任务批量检测 |
| **动态Agent注册** | AgentRegistry管理所有注册Agent，支持多实例部署（instance_id区分），心跳超时自动触发注销事件，事件驱动架构（Spring ApplicationEvent） |

### 3.3 数据库设计

| 亮点描述 | 具体说明 |
|---------|---------|
| **MyBatis-Plus** | 使用MyBatis-Plus 3.5.6，简化CRUD操作，支持分页插件、条件构造器、逻辑删除 |
| **表设计规范** | 不使用外键约束（应用层保证一致性）、下划线命名、TINYINT状态枚举、datetime时间字段、is_deleted逻辑删除 |
| **核心表结构** | ai_agent_info（Agent信息）、ai_agent_registration（注册表）、ai_workflow（编排配置）、ai_workflow_execution（执行记录）、ai_a2a_task（A2A任务） |

### 3.4 前沿技术应用

| 亮点描述 | 具体说明 |
|---------|---------|
| **流式输出（SSE）** | 使用Spring WebFlux实现Server-Sent Events流式输出，TTS Agent支持实时返回音频数据片段，降低感知延迟 |
| **Redis Stream** | 使用Redis Stream作为A2A消息中间件，XREADGROUP消费组模式保证消息顺序，支持消息持久化和重消费 |
| **JWT认证** | 使用JWT实现无状态认证，支持Token生成、验证、过期机制，Token中存储用户ID和角色信息 |

### 3.5 性能与可靠性

| 亮点描述 | 具体说明 |
|---------|---------|
| **多实例部署支持** | 通过instance_id区分不同实例，Redis分布式锁保证心跳检测互斥性，避免多实例冲突 |
| **超时控制与降级** | 对外部服务调用设置超时时间，捕获异常并返回友好提示，防止故障蔓延 |
| **事件驱动架构** | 使用Spring ApplicationEvent实现注册/注销/心跳超时事件通知，解耦核心逻辑 |

### 3.6 前端技术栈

| 亮点描述 | 具体说明 |
|---------|---------|
| **Vue3 + Vite** | 采用Vue3组合式API（Composition API）和Vite构建工具，提升开发效率和运行时性能 |
| **ECharts图谱** | 使用ECharts实现Agent调用关系图谱，支持节点拖拽、缩放、流动动画等交互效果 |
| **Element Plus** | 使用Element Plus组件库，快速构建管理后台页面 |

### 3.7 部署与运维

| 亮点描述 | 具体说明 |
|---------|---------|
| **Docker Compose** | 使用Docker Compose编排MySQL、Redis、Nginx、RocketMQ等服务，一键启动完整环境 |
| **Nginx配置** | 配置Nginx反向代理和静态资源缓存，优化前端加载速度 |
| **GitHub Actions CI/CD** | 配置GitHub Actions自动化CI流程，包含代码构建、测试执行等环节 |
